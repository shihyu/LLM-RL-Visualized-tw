第2部分：大型模型基礎

【LLM基礎】LLM（Large Language Model）結構

  • LLM主要有Decoder-Only（僅解碼器）或MoE（Mixture of Experts, 專家混合模型）兩種形式，兩者在整體架構上較為相似，主要區別為MoE在FFN（前饋網路）部分引入了多個專家網路。
  • 一個典型的LLM結構，可分為三部分——輸入層、多層Decoder堆疊結構和輸出層（包括語言模型頭與解碼模組）

【LLM基礎】LLM生成與解碼（Decoding）過程

  • 解碼策略是決定最終輸出文字流暢性、多樣性及整體表現的核心因素。常見的解碼演算法包括：貪婪搜尋、波束搜尋（Beam Search）及其衍生演算法、多項式取樣、Top-K取樣、Top-P取樣、對比搜尋（Contrastive Search）、投機解碼（Speculative Decoding）、前瞻解碼（Lookahead Decoding）、DoLa解碼等。
  • LLM的輸出層負責根據機率分佈，應用解碼演算法確定最終預測的下一個/N個詞元。
  • 根據機率分佈，應用解碼策略（例如隨機取樣或選擇最大機率）確定最終預測的下一個詞元。例如，在貪婪搜尋解碼策略下，選擇機率最高的詞元“我”作為預測結果。
  • 每一個token（詞元）的生成都需要重新走一遍所有層Transformer結構。
  • 這裡只展示了每次預測一個詞元的方案，也有一次性預測多個詞元的方案，詳見《大型模型演算法：強化學習、微調與對齊》第四章。

【LLM基礎】LLM輸入層

  • LLM的輸入層將輸入文字轉換為多維數值矩陣（Tensor，張量），以便送往模型主體結構進行計算。

【LLM基礎】LLM輸出層

LLM的輸出層負責根據隱藏狀態（多維數值矩陣）預測下一個詞元（文字）。處理流程如圖所示：

  • （1）輸入隱藏狀態：Decoder最後一層的隱藏狀態（Hidden States）作為LLM輸出層的輸入。如圖中所示維度為3×896的數值矩陣，包含了字首序列的所有語義資訊。
  • （2）語言模型頭（LM Head）：通常是一個全連線層，用於將隱藏狀態轉換為Logits（推理時只計算最後一個位置的Logits）。如圖所示，Logits的形狀為3×151936，對應於詞表中每個詞元的得分。
  • （3）提取最後位置的Logits：預測下一個詞元僅依賴於字首序列最後一個位置的Logits，因此需要從所有位置的Logits中提取最後一個位置的Logits。如圖所示，提取得到一個151936維的向量 [2.0, 3.1, −1.7, ... , −1.7]，用於後續預測。
  • （4）轉換為機率分佈（Softmax）：透過Softmax函式將Logits轉換為機率分佈，得到詞表中每個詞元的機率。例如，生成的151936維機率向量為[0.01, 0.03, 0.001, ... , 0.001]，該向量內所有機率之和為1。機率值越大，表示該詞元作為下一個詞元的可能性越高。例如，“我”的機率為0.34。
  • （5）解碼（Decoding）：根據機率分佈，應用解碼策略（例如隨機取樣或選擇最大機率）確定最終預測的下一個詞元。例如，在貪婪搜尋解碼策略下，選擇機率最高的詞元“我”作為預測結果。

【LLM基礎】多模態模型結構（VLM、MLLM ...）

多模態模型根據側重不同，通常有以下多種稱呼：

  • VLM（Vision-Language Model，視覺語言模型）
  • MLLM（Multimodal Large Language Model，多模態大型語言模型）
  • VLA（Vision-Language-Action Model，視覺語言動作模型）

【LLM基礎】LLM訓練流程

  • 大型模型的訓練主要分為兩個階段——預訓練（Pre-Training）和後訓練（Post-Training）。在每個階段，所使用的訓練資料、訓練正規化（演算法）、訓練目標和超引數均有所不同。
  • 預訓練階段包括初期訓練（基於海量資料的短上下文訓練）、中期訓練（長文字/長上下文訓練）以及退火（Annealing）訓練等。此階段以自監督學習為主，使用的資料量最大，同時也是最消耗算力的環節。
  • 後訓練階段則包含多種可選的訓練正規化（演算法），包括但不限於SFT（監督微調）、蒸餾、RSFT（拒絕取樣微調）、RLHF（基於人類反饋的強化學習）、DPO（直接偏好最佳化），以及其他強化學習方法，例如GRPO（群體相對策略最佳化）、PPO（近端策略最佳化）等。其中，某些環節也可進行多輪迭代訓練，例如多輪拒絕取樣微調（Rejection Sampling Fine-Tuning, RSFT）。

第3部分：SFT（有監督微調）

【SFT】微調（Fine-Tuning）技術分類

  • 可用於SFT的微調技術種類多樣，如下圖的分類圖所示：前兩種方法僅需基於預訓練模型主體進行微調，開發成本較低；而並聯低秩微調和Adapter Tuning則需要引入額外的新模組，實施過程相對複雜一些。這些方法均是針對模型引數進行微調，而基於Prompt的微調則另闢蹊徑，從模型的輸入著手進行微調。

【SFT】LoRA（1 of 2）

  • LoRA（Low-Rank Adaptation, 低秩適配微調）由微軟的研究團隊於2021年發表。由於其高效的微調方式和良好的效果，在各類模型中得到廣泛的應用。LoRA的核心思想在於——微調前後模型的引數差異∆W具有低秩性。
  • 一個矩陣具有低秩性，意味著它包含較多的冗餘資訊，將其分解為多個低維矩陣後，不會損失過多有用資訊。例如，如圖所示，一個1024×1024的矩陣可以被近似分解為一個1024×2矩陣與一個2×1024矩陣的乘積，從而將引數量約減少至原來的0.4%。

【SFT】LoRA（2 of 2）

  • A和B的初始化方式：
  • （1）矩陣A使用隨機數初始化，如kaiming初始化；
  • （2）矩陣B使用全0初始化，或使用極小的隨機數初始化。
  • 目的在於——確保在訓練初期，插入的LoRA模組不會對模型整體輸出造成過大的擾動。

【SFT】Prefix-Tuning

  • Prefix-Tuning由斯坦福大學的研究團隊提出，可以用於對語言模型進行輕量級微調。
  • 如圖所示，該方法在輸入的起始位置插入一段連續的、可訓練的向量（Embedding），稱為字首（Prefix）。在處理後續的Token時，Transformer將這些向量視為 虛擬Token，並參與Attention計算。

【SFT】TokenID與詞元的對映關係

  • 以SFT資料（經過ChatML格式預處理）為例，該資料經過Tokenizer處理後，被轉換為33個Token，對應33個序列位置，如圖所示。
  • 每個Token ID與相應的詞元一一對應。

【SFT】SFT的Loss（交叉熵）

  • 與預訓練階段類似，SFT的Loss也是基於交叉熵（CE，Cross Entropy）

【SFT】指令資料的來源

  • 指令資料（Instructions）是指為模型提供的一組引導性輸入及期望輸出的描述，通常包括問題（或任務提示）及其對應的答案，常用於對模型進行微調訓練。

【SFT】多個數據的拼接（Packing）

  • 模型訓練時通常使用固定長度的輸入。當輸入的資料長度不一致時，對於短序列，會在其末尾填充（padding）以匹配最大序列長度，這會導致計算資源的浪費。
  • 因此，正如圖所示，常見的做法是將多條資料拼接（Packing）在一起，填充到一個固定長度的輸入序列中。
  • 為了確保計算過程中不同資料之間互不干擾，通常需要重設位置編號（Reset position ID）和注意力掩碼（Reset attention mask），以在計算Attention時保持各條資料的語義獨立性。

第4部分：DPO（直接偏好最佳化）

【DPO】RLHF與DPO的訓練架構對比

不同於RLHF，DPO以監督學習的訓練方式，大幅簡化了對齊訓練：

  • 流程簡潔：DPO直接對策略模型進行最佳化，不需要預先訓練Reward模型（獎勵函式）。DPO只需要基於預先給定的偏好資料進行訓練，無需中途取樣。
  • 穩定性：DPO是一種監督學習方法，擺脫了強化學習訓練的不穩定性。
  • 低開銷：DPO在訓練過程中只需要載入一個模型（只需載入策略模型，而對於參考模型，可以將參考模型的輸出結果預先錄製好，然後在訓練時就不需要載入），算力開銷更低，更易於落地實踐。

【DPO】Prompt的收集

【DPO】DPO（Direct Preference Optimization）

  • DPO（Direct Preference Optimization，直接偏好最佳化）是由斯坦福大學等研究團隊於2023年提出的一種偏好最佳化演算法，可用於LLM、VLM與MLLM的對齊訓練。
  • 該演算法在基於PPO的RLHF基礎上進行了大幅簡化。
  • DPO演算法跳過了訓練獎勵模型這一中間過程，直接（Direct）最佳化策略模型 ——這正是DPO命名中“D（Direct）”的含義所在。 DPO的訓練涉及2個模型——策略模型和參考模型。它們的初始化方法如下：
  • 策略模型：直接複製SFT模型作為初始化。
  • 參考模型：通常也從SFT模型複製。但在某些情況下，可能會選擇從一個比SFT模型更強的模型進行復制。此時，需特別關注參考模型與策略模型的匹配性，主要涉及兩者的KL距離及訓練資料分佈等方面。

【DPO】DPO訓練全景圖

  • DPO訓練時，可以選擇載入2個模型（策略模型 和參考模型 ），也可以只加載1個模型（策略模型 ）。本文將優先分析載入2個模型的情況。
  • DPO的整體訓練流程如圖所示（藍色色塊代表偏好資料對中的“優質回答” 及其對應的中間計算結果；粉色色塊代表偏好資料對中的“劣質回答” 及其對應的中間計算結果）。

【DPO】β引數對DPO的影響

  • 在DPO中， 引數的作用類似於其在RLHF中的作用。

【DPO】隱式獎勵差異對引數更新幅度的影響

  • DPO的梯度更新旨在增加優質回答的機率，同時減少劣質回答的機率。
  • 更重要的是，梯度中包含一個動態係數——優質和劣質回答的隱式獎勵差異。換言之，這個動態係數反映了隱式“獎勵模型”在對偏好順序的判斷上有多大誤差。

第5部分：免訓練的大型模型最佳化技術

【免訓練的最佳化技術】CoT（Chain of Thought）與傳統問答的對比

  • CoT（Chain of Thought，思維鏈）由Jason Wei等研究者於2022年在Google期間提出，是大型模型領域的一項重要技術創新，並迅速被廣泛應用於學術研究和實際場景。
  • 其核心思想是透過顯式地分步推理，提升模型在複雜推理任務中的表現。

【免訓練的最佳化技術】CoT、Self-consistency CoT、ToT、GoT [87]

  • 在CoT展現其潛力後，迅速衍生出多種相關技術，例如ToT、GoT、Self-consistency CoT、Zero-shot-CoT、Auto-CoT、MoT、XoT等

【免訓練的最佳化技術】窮舉搜尋（Exhaustive Search）

  • Token的生成過程可以形象地表示為一個以詞表大小 V=105 為基數的多叉樹結構。理論上，採用窮舉搜尋（Exhaustive Search）的方法能夠獲得全域性最優解。然而，窮舉搜尋的計算代價極高。

【免訓練的最佳化技術】貪婪搜尋（Greedy Search）

  • 貪婪搜尋（Greedy Search）在生成下一個Token（詞元）時，每次都會選擇當前機率最高的一個Token，不考慮生成序列的全域性最優性或多樣性，然後繼續對下一個Token位置執行相同的操作。儘管這種方法簡單快速，但生成的內容可能過早陷入區域性最優，缺乏多樣性。

【免訓練的最佳化技術】波束搜尋（Beam Search）

  • Beam Search（波束搜尋）在每一步生成時，不僅僅選擇一個最優Token（詞元），而是保留多個候選序列（稱為Beam，即波束），其餘的路徑則被剪枝。這些候選序列會在後續步驟中繼續擴充套件，直到生成結束。
  • 最終，從所有候選序列中選擇得分最高的一條作為最終輸出。Beam的數量（num_beams引數）越大，搜尋空間越廣，生成結果越接近全域性最優，但計算成本也隨之增加。

【免訓練的最佳化技術】多項式取樣（Multinomial Sampling）

  • 多項式取樣（Multinomial Sampling）是生成式模型中一種常見的隨機取樣方法，生成下一個Token時，以模型預測的機率分佈為依據，在機率分佈中“按機率大小”隨機抽取Token（而非等機率隨機抽樣）。
  • 包含Top-K、Top-P等

【免訓練的最佳化技術】Top-K取樣（Top-K Sampling）

  • Top-K取樣（Top-K Sampling）是一種在生成任務中常用的策略，類似於多項式取樣，但其取樣候選池經過限制。使用Top-K取樣時，每一步生成Token時僅保留模型預測機率最高的前K個詞，並從中按機率分佈進行隨機抽樣。

【免訓練的最佳化技術】Top-P取樣（Top-P Sampling）

  • Top-P取樣（Top-P Sampling），又稱核取樣（Nucleus Sampling），該方法透過動態選擇一個最小候選集合，使得候選詞的機率和達到設定的機率閾值P，然後，在該候選集合中隨機取樣。與Top-K取樣相比，Top-P取樣能夠根據機率累積動態調整候選集的大小。

【免訓練的最佳化技術】RAG（檢索增強生成,Retrieval-Augmented Generation）

  • RAG（Retrieval-Augmented Generation，檢索增強生成）是一種結合資訊檢索與模型生成的技術，透過引入外部知識庫或檢索系統，增強生成式模型的知識範圍和回答準確性。
  • Meta（前身為Facebook AI Research）等研究團隊於2020年在其發表的工作中提出了RAG，並顯著提升了知識密集型NLP任務的效能。
  • RAG的原理如圖所示，整體可分為兩部分——離線構建環節和線上服務環節。

【免訓練的最佳化技術】功能呼叫（Function Calling）

  • 功能呼叫（Function Calling），也稱工具呼叫（Tool Use），是指在基於大型模型完成任務的過程中，Agent透過特定機制呼叫外部物件，獲取返回結果後將其與原始Prompt一起輸入到大型模型，由大型模型進一步推理並完成特定任務。

【強化學習基礎】強化學習(Reinforcement Learning, RL)的發展歷程

  • 強化學習可以追溯至20世紀50年代，之後由Richard S. Sutton等學者相繼提出了許多關鍵演算法和理論。
  • 自2012年起，隨著深度學習的迅猛發展，強化學習在多個領域催生了備受關注的應用。
  • 2022年11月，結合RLHF（基於人類反饋的強化學習）訓練的ChatGPT正式釋出。
  • 隨後在2024年12月，OpenAI推出經過更深入強化學習最佳化的o1模型，這極大地激發了業界基於強化學習對大型模型進行訓練的關注和投入。

【強化學習基礎】三大機器學習正規化

  • 無監督學習、監督學習和強化學習並列為三大機器學習正規化。

【強化學習基礎】強化學習的基礎架構

  • 強化學習的執行主要涉及兩個核心角色：智慧體（Agent）和環境（Environment）。
  • 智慧體透過感知環境的狀態，基於策略選擇並執行動作；
  • 環境接收動作後更新狀態並反饋獎勵訊號。

【強化學習基礎】探索與利用問題（Exploration and Exploitation）

  • 探索與利用（Exploration-Exploitation）的平衡是強化學習的核心挑戰之一。
  • 智慧體在學習過程中需要在“嘗試新動作”（探索）與“選擇當前已知的最優動作”（利用）之間進行權衡。過度探索可能導致學習過程緩慢，甚至無法收斂到最優解；而過度利用則可能使智慧體陷入區域性最優，無法達到全域性最優回報。

【強化學習基礎】Ɛ-貪婪策略下使用動態的Ɛ值

  • Ɛ-貪婪（Epsilon Greedy）是一種簡單而有效的策略，用於在強化學習中平衡探索與利用。透過引入探索機率因子Ɛ，控制探索與利用的比例。
  • 在訓練初期，可以將Ɛ的值設得比較大，隨著訓練的進行，逐步降低 Ɛ的值，使智慧體在初期更多地進行探索，隨著對環境的瞭解加深，逐漸轉向利用，從而提高學習效率和最終策略的效能。

【強化學習基礎】強化學習訓練正規化的對比

  • 同策略（On-policy）、異策略（Off-policy）、線上強化學習（Online RL）和離線強化學習（Offline RL）
  • 根據經驗收集與策略訓練的方式，主要可以劃分為四個核心概念：同策略（On-policy）、異策略（Off-policy）、線上強化學習（Online RL）和離線強化學習（Offline RL）。
  • 同策略（On-policy）：當行為策略與目標策略相同時，稱為同策略學習。也就是說，智慧體根據當前策略選擇動作，並利用這些經驗來更新當前策略，兩者是同一個版本的策略。如圖所示，在同策略下，行為策略和目標策略都是πn。本質上是同一個策略（模型），為了便於理解和對比，圖中繪製了兩個策略的示意圖。例如，策略梯度方法和SARSA演算法都是典型的同策略演算法。
  • 異策略（Off-policy）：當行為策略與目標策略不同時，稱為異策略學習。智慧體可以使用某種策略進行探索，但學習（訓練）和最佳化的是另一種策略，兩者屬於不同版本（或不同型別）的策略。例如，Q-learning、DQN等演算法是典型的異策略演算法，儘管可能使用Ɛ-貪婪策略進行探索，但最終學習的是最優策略。
  • 線上強化學習（Online RL）：在訓練過程中，策略持續與環境互動並收集經驗，訓練資料來源源不斷地動態生成，目標策略也在透過訓練不斷進化。
  • 離線強化學習（Offline RL）：又稱批次強化學習（Batch RL），有時也稱為完全異策略強化學習（Fully Off-policy RL）。在訓練過程中，智慧體完全不與環境互動，而是在一個與線上環境隔離的環境中進行訓練，僅依賴預先收集的資料。這些資料可能來源於歷史記錄、其他策略的積累，或是模擬環境生成的樣本。

【強化學習基礎】獎勵（Reward）、回報（Return）、價值（Value）的關係

  • 獎勵（Reward）、回報（Return）和價值（Value）三者的關係如圖所示。需要注意的是，此處假設折扣因子γ=1。
  • （1）獎勵（Reward）：即時獎勵，是在某一狀態下獲得的區域性收益或短期收益。
  • （2）回報（Return）：未來回報，是從當前狀態st開始計算，之後所有獎勵的累積和，即未來的總收益。

  • 狀態價值函式（State-Value Function），記為Vπ(s)，表示在狀態s下，遵循策略π時，智慧體未來能夠獲得的回報的期望值。

【強化學習基礎】蒙特卡洛（Monte Carlo，MC）法預估狀態St的價值

  • 蒙特卡洛（Monte Carlo, MC）方法是一種基於隨機取樣的計算方法，利用隨機取樣來解決在原則上可能是確定性的難題。
  • 在強化學習中，蒙特卡洛方法是指透過多次實際執行試驗，獲得多個實際的執行軌跡，根據這些結果來估計價值函式或最佳化策略，特別適用於那些環境模型未知或難以解析的問題。

【強化學習基礎】DQN的“高估”問題

  • DQN演算法的兩大核心問題：（1）“高估”問題; （2）“狗追尾巴”問題。
  • （1）高估問題：DQN在選擇和評估動作時，動作價值Q常被過高估計，導致策略優化出現偏差，影響最終效能。這主要由於最大化操作中包含了高估部分的值，如圖所示，最終目標價值yt中包含了額外高估的部分，並且這種高估會逐步累積。此外，這種高估是“非均勻”的，即不同動作的價值Q被高估的程度各不相同。
  • （2）狗追尾巴問題：也稱為自舉問題，如式所示，DQN的訓練過程可以看做一個“迴歸問題”，但是迴歸（要擬合）的目標yt總是在變，所以提升了訓練的難度。在DQN中，計算目標價值 和當前狀態st的價值預測值 使用相同的網路權重θ。由於迴歸的目標yt依賴於當前網路的權重θ，當網路權重更新後變為θ′；下一步訓練時，計算出的價值預測值和迴歸的目標yt值將同步變化。這種動態相互依賴關係就像“狗追尾巴”一樣，不斷迴圈，導致最佳化過程容易不穩定，甚至難以收斂。

【強化學習基礎】基於價值vs基於策略（Value-Based vs Policy-Based）

  • 根據學習物件的不同：分為基於價值（Value-Based）、基於策略（Policy-Based）以及演員-評委（Actor-Critic）方法。
  • 基於價值的演算法透過估計各狀態的價值函式（例如V或Q）來間接推匯出最優策略；
  • 基於策略的演算法直接學習和最佳化策略函式π；
  • 演員-評委方法則同時結合了基於價值和基於策略的特點。

【強化學習基礎】策略梯度（Policy Gradient）

  • 策略梯度是強化學習眾多演算法的理論基石。例如，PPO演算法、GRPO演算法、DPG演算法以及基於Actor-Critic架構的眾多演算法都建立於策略梯度之上。
  • RL之父 Richard S. Sutton等人則在前人基礎上進一步提出了更為嚴謹且系統化的策略梯度定理（Policy Gradient Theorem）。
  • 與基於價值（Value-Based）的方法（例如DQN等）不同，透過直接最佳化策略以最大化回報（累積獎勵），直接對策略函式進行引數化，並透過梯度上升或下降的方式最佳化策略引數。

【強化學習基礎】多智慧體強化學習（MARL，Multi-agent reinforcement learning）

  • 多智慧體強化學習（Multi-Agent Reinforcement Learning，MARL）是強化學習的一個重要分支，旨在研究多個智慧體在共享環境中透過互動（包括競爭與合作）學習最優策略的問題。
  • 例如，DeepMind的研究團隊開發的AlphaGo、AlphaZero和AlphaStar等，OpenAI開發的OpenAI Five，展示了多智慧體強化學習的強大潛力。

【強化學習基礎】模仿學習（IL，Imitation Learning）

  • 模仿學習（Imitation Learning，IL）是一種透過觀察並模仿專家行為來學習複雜任務的機器學習方法。
  • 與傳統的強化學習不同，模仿學習無需明確的獎勵函式，而是直接從專家的示範中學習策略。這種方法特別適用於那些難以手工設計獎勵函式的任務場景。 常見的模仿學習演算法主要分為三類：
  • （1）行為克隆（Behavioral Cloning，BC）：基於監督學習方法，直接從專家的狀態-動作對中學習對映關係。
  • （2）逆向強化學習（Inverse Reinforcement Learning，IRL）：基於強化學習方法，透過推斷獎勵函式間接學習策略。
  • （3）生成對抗模仿學習（Generative Adversarial Imitation Learning，GAIL）：借鑑生成對抗學習的思想，透過對抗過程最佳化策略。

【強化學習基礎】行為克隆（BC，Behavior Cloning）

  • 行為克隆（Behavioral Cloning,BC）是一種結合了監督學習的機器學習方法，透過學習專家在不同狀態（State）下采取的動作（Action）的對映關係來模仿專家的行為。
  • 行為克隆將問題視為一個迴歸或分類問題，輸入為環境的狀態，輸出為相應的動作。透過最小化預測動作與專家動作之間的誤差，模型能夠在類似狀態下采取與專家相似的動作。

【強化學習基礎】逆向強化學習（IRL，Inverse RL）、強化學習（RL）

  • 逆向強化學習（Inverse Reinforcement Learning, IRL）指的是根據專家或智慧體的行為，推斷其背後的獎勵函式，然後，基於此獎勵函式學習最優策略。
  • 這一過程與傳統強化學習（RL）相反，後者通常是在已知獎勵函式的情況下學習最優策略，而逆向強化學習則是在已知行為的基礎上反推出獎勵函式，再進一步學習策略。
  • 吳恩達（Andrew Y. Ng）和Stuart Russell於2000年發表了《Algorithms for Inverse Reinforcement Learning》一文，系統性地闡述了逆強化學習的核心概念。

【強化學習基礎】有模型（Model-Based）、無模型（Model-Free）

  • 根據是否依賴環境模型：分為有模型（Model-Based）和無模型（Model-Free）的方法。
  • 有模型的方法利用環境模型進行規劃和決策；無模型的方法則直接透過與環境互動學習策略或價值函式。

【強化學習基礎】封建等級強化學習（Feudal RL）

  • 分層強化學習（Hierarchical Reinforcement Learning, HRL）是一種分而治之的強化學習框架，透過將複雜的任務分解為多個層次的較小的子任務或子策略，使智慧體能夠在不同的抽象層次上學習和決策。
  • 該領域發展至今，已經湧現出多種經典演算法，例如，封建等級強化學習（Feudal RL）和MAXQ等演算法。
  • 封建等級強化學習（Feudal Reinforcement Learning, Feudal RL）最早由AI巨佬Geoffrey E. Hinton等人提出，Hinton因其在人工智慧領域的突出貢獻於2024年獲得諾貝爾物理學獎。

【強化學習基礎】分佈價值強化學習（Distributional RL）

  • 分佈價值強化學習（Distributional Reinforcement Learning）指的是直接對回報（累積獎勵）的分佈進行建模，而不僅僅是對其期望值進行估計。
  • 分佈價值模型包含了更豐富的價值資訊，能夠更全面地捕捉環境的不確定性，從而在策略最佳化和潛在風險管理等方面提供優勢。

第7部分：策略最佳化架構及衍生演算法

【策略最佳化架構演算法及其衍生】Actor-Critic架構

  • Actor-Critic架構（AC架構），即演員-評委架構，是一種應用極為廣泛的強化學習架構。知名的PPO、DPG、DDPG、TD3等演算法均基於Actor-Critic架構。
  • （1）Actor（演員）：對應於策略模型π，負責選擇動作，直接輸出策略π(a | s)，即在給定狀態s下選擇動作a的機率分佈。
  • （2）Critic（評委）：對應於價值模型Q，評估Actor執行的動作的好壞，這可以協助Actor逐步最佳化策略模型的引數。
  • 該架構有效融合了基於策略（Policy-Based）和基於價值（Value-Based）的方法，結合了兩種方法的優勢，透過同時學習策略和價值函式來提升學習效率與穩定性，是一種混合型方法。

【策略最佳化架構演算法及其衍生】引入基線與優勢（Advantage）函式A的作用

  • 基礎版本的Actor-Critic架構存在高方差等問題。為了解決這些問題，A2C（Advantage Actor-Critic）方法在Actor-Critic的基礎上引入基線（Baseline），並進一步構造了優勢函式（Advantage Function）。
  • （1）引入基線：基線通常採用狀態價值函式V(s)，即在狀態s下的預期回報。將基線設為V(s)，相當於在每個狀態s下設定了一個平均水平。
  • （2）構造優勢函式：優勢函式衡量在給定狀態下，採取某一動作相對於“平均水平”的優劣，即某個動作a相對於特定狀態下其他動作的相對優勢。優勢函式關注的是執行某個動作的相對優勢，而非動作的絕對價值。

【策略最佳化架構演算法及其衍生】GAE（廣義優勢估計,Generalized Advantage Estimation）演算法

  • GAE（Generalized Advantage Estimation, 廣義優勢估計）演算法由RL巨佬John Schulman等發表。該演算法是PPO等演算法的關鍵組成部分。
  • GAE演算法借鑑了TD(λ)演算法的思路，（TD(λ)如《大型模型演算法》5.3.3節所述）。TD(λ)演算法透過調節λ因子在偏差和方差之間取得平衡，GAE演算法也使用λ因子以達到類似的目的。
  • GAE演算法計算時，通常採用遞迴求解的方式進行。
  • GAE演算法的虛擬碼如下。演算法 6.1 GAE 演算法核心實現
import numpy as np

def compute_gae(rewards, values, gamma=0.99, lambda_=0.95):
    """
    引數:
        rewards (list 或 np.ndarray): 每個時間步收集到的獎勵 r，形狀為 (T,)
        values  (list 或 np.ndarray): 每個狀態的價值估計 V，形狀為 (T+1,)
        gamma   (float): 折扣因子 γ
        lambda_ (float): GAE 的衰減引數 λ
    返回:
        np.ndarray: 優勢估計 A，形狀為 (T,)。例如對於 T=5，A=[A0, A1, A2, A3, A4]
    """
    T = len(rewards)            # 時間步數 T，終止時間步為 t=T-1
    advantages = np.zeros(T)    # 優勢估計陣列，例如 [A0, A1, A2, A3, A4]
    gae = 0                     # 初始化 GAE 累計值為 0

    # 反向從時間步 t=T-1 到 t=0 進行迭代計算，總共迭代 T 次
    for t in reversed(range(T)):
        # δ_t = r_t + γ * V(s_{t+1}) - V(s_t)
        delta = rewards[t] + gamma * values[t + 1] - values[t]
        # A_t = δ_t + γ * λ * A_{t+1}
        gae = delta + gamma * lambda_ * gae
        advantages[t] = gae

    return advantages

  • PPO（Proximal Policy Optimization，近端策略最佳化），有多種變體，例如PPO-Penalty和PPO-Clip，這些演算法繼承了部分TRPO演算法的思想。
  • PPO-Clip因其更優的效果而獲得了更多關注和應用，因此通常所說的PPO即指PPO-Clip。

【策略最佳化架構演算法及其衍生】TRPO（Trust Region Policy Optimization）及其置信域

  • TRPO（Trust Region Policy Optimization，置信域策略最佳化），可以說是PPO的前身。
  • 該演算法是對策略梯度演算法的改進，基於兩個核心概念：置信域和重要性取樣（Importance Sampling）。雖然這兩個概念並非在TRPO中首次提出，但TRPO將它們與策略梯度演算法相結合，顯著提升了演算法的效果。
  • TRPO的核心思想是在最大化目標函式J(θ)的同時，限制新舊策略之間的差異。

【策略最佳化架構演算法及其衍生】重要性取樣（Importance sampling）

  • 重要性取樣在諸如TRPO和PPO等強化學習演算法中具有關鍵作用，其主要功能是修正新舊策略之間的分佈差異，使得可以利用舊策略所採集的資料來最佳化新策略。
  • 重要性取樣（Importance Sampling）是一種基於蒙特卡洛取樣思想的方法，經常被用於估計期望值和積分。該方法基於輔助分佈進行取樣，並透過重要性權重對估計值進行修正，從而提高取樣和估計的效率。
  • 重要性取樣需滿足該條件：如果在某個x上p(x)的機率值大於0，那麼p'(x)在同一個位置的機率也必須大於0（p'(x)不能為零，意味著該位置在p'(x)中也有可能性）。這是因為如果在p(x)非零的地方p'(x)= 0，那麼在這些x上的貢獻就無法透過p'(x)來取樣計算，一些x的取值會在積分中丟失。

  • GRPO（Group Relative Policy Optimization, 群體相對策略最佳化）是一種基於策略的強化學習演算法，由DeepSeek團隊提出，並已在DeepSeek、Qwen等模型的訓練中得到應用。
  • 傳統的PPO方法除了訓練策略模型外，還需額外構建一個規模相近的價值網路，這會顯著增加計算和視訊記憶體的開銷。
  • 如圖所示，GRPO摒棄了單獨的價值網路，並透過多項改進，在保留PPO核心思想的基礎上，顯著降低了訓練所需資源，同時確保了策略更新的高效性和穩定性。
  • GRPO的核心思想在於利用群體相對優勢估計來取代傳統的價值模型。具體來說，GRPO透過取樣一組候選輸出，並將這些輸出的平均獎勵作為基線，來計算各個輸出的優勢值。這種方法不僅避免了對額外價值模型的依賴，同時也充分發揮了獎勵模型的比較特性，從而提高了訓練的效率和穩定性。

【策略最佳化架構演算法及其衍生】確定性策略vs隨機性策略（Deterministic policy vs. Stochastic policy）

  • 強化學習的策略型別可以分為：確定性策略（Deterministic Policy）和隨機性策略（Stochastic Policy）。
  • 確定性策略：在每個狀態下，策略都會輸出一個確定的動作。
  • 隨機性策略：在每個狀態下，策略輸出的是動作的機率分佈。

【策略最佳化架構演算法及其衍生】確定性策略梯度（DPG）

  • 確定性策略梯度（Deterministic Policy Gradient, DPG）演算法由RL大佬David Silver在DeepMind期間與其他研究者於2014年發表的論文中系統性闡述，該演算法旨在解決連續動作空間中的強化學習問題。
  • DPG採用確定性策略，將狀態直接對映到具體的動作。
  • DPG採用了演員-評委（Actor-Critic）架構。

【策略最佳化架構演算法及其衍生】DDPG（Deep Deterministic Policy Gradient）

  • 深度確定性策略梯度（Deep Deterministic Policy Gradient, DDPG）是確定性策略梯度（DPG）演算法的改進版本，結合了深度Q網路（DQN）的思想，由DeepMind的研究團隊發表。
  • DDPG在訓練過程中需要載入4個模型——包括2個策略網路（Actor）和2個評估網路（Critic）。
  • 知名的TD3 ：雙延遲深度確定性策略梯度（Twin Delayed Deep Deterministic Policy Gradient, TD3）演算法是對DDPG（Deep Deterministic Policy Gradient）的顯著改進。在TD3的訓練過程中，需要載入6個模型——包括2個策略網路（Actor）和4個評估網路（Critic）。這些網路分工明確、相互配合，透過延遲更新目標網路等技巧，有效提升了訓練的穩定性和效能。

【RLHF與RLAIF】語言模型的強化學習建模

  • 大型模型（LLM、MLLM、VLM等）為何可以用RL正規化進行訓練？ ——這可能是大型模型RL方向最基礎的一個問題——如何將大型模型的生成過程建模為一個強化學習問題？ 如圖所示，以下是生成式語言模型中的一些關鍵概念在強化學習正規化下，具體的定義如下：
  • （1）動作（Action, A）：動作指模型在每個時間步生成的一個Token。例如，當模型從詞表中選擇下一個輸出詞時，這一選擇就是一個動作。
  • （2）動作空間（Action Space）：動作空間是語言模型的詞表（Vocabulary），即所有可能的Token的集合，通常包含約10萬級的詞元。
  • （3）智慧體（Agent）：智慧體是負責決策的實體，即LLM。它負責接受文字輸入並生成一系列文字或其機率分佈。
  • （4）策略（Policy）：策略是LLM對應的模型引數，決定了模型生成文字的具體規則和行為。
  • （5）狀態（State, S）：狀態是模型當前的輸入上下文，通常由已生成的Token序列構成。例如，在針對Prompt為“你是誰？”的生成過程中，狀態會隨著生成的Token動態更新，形成如“你是誰？”、“你是誰？我”、“你是誰？我是”等狀態。
  • （6）回合（Episode）：回合是一次完整的文字生成過程，從初始輸入（例如一個問題或指令）開始，到模型生成結束（通常以特殊符號或表示）。
  • （7）價值（Value, V）：價值表示在特定狀態下或採取特定動作後，模型未來所能獲得的預期回報。它衡量當前動作（Token或詞）的選擇質量及其後續可能動作的潛在影響，通常由單獨的價值網路進行估計。

【RLHF與RLAIF】RLHF的兩階段式訓練流程

RLHF訓練可以分為兩個階段：

  • （1）階段一（SFT，訓練獎勵模型）：根據使用者輸入（Prompt），生成多種候選回答，並由人工對這些回答進行偏好標註。這些標註資料作為訓練樣本，用於監督學習訓練獎勵模型，使其能夠預測不同回答的偏好得分。
  • （2）階段二（RL，基於PPO的強化學習）：聯合策略模型、參考模型、獎勵模型和價值模型，基於PPO等RL演算法進行強化學習訓練。透過迭代最佳化，策略模型逐步提升生成內容的質量，更加符合人類的需求和價值觀。 兩個階段的樣本如下：
  • （1）獎勵模型樣本：獎勵模型的訓練依賴於偏好資料集（Prompt + 回答），其構建和清洗過程詳見《大型模型演算法》3.2.1節和3.2.3節。
  • （2）PPO訓練的樣本：該階段的訓練樣本僅需要Prompt即可，無需回答。如《大型模型演算法》5.1.1節所述，強化學習透過智慧體與環境的互動進行學習，無需明確的標籤（回答）。

  • 獎勵模型（Reward Model, RM）在RLHF訓練中起著關鍵作用，與價值模型共同指導策略模型的最佳化方向。
  • 獎勵模型的核心作用是為特定Prompt對應的回答結果輸出一個獎勵分數，用於指導策略模型的最佳化。 通常，獎勵模型是在經過SFT的LLM基礎上進行小幅改造而來。由於SFT之後的模型已具備豐富的語言處理能力，直接複用其底層的Transformer結構可以顯著減少訓練時間和資料需求。改造過程如圖所示：
  • （1）共享底層結構：獎勵模型複用了LLM的底層結構及其權值，包括所有的Transformer Decoder層和Embedding層，從而確保兩者具備一致的隱藏狀態表示能力。這種共享設計不僅降低了訓練複雜度，還顯著減少了從零訓練所需的計算資源和資料需求。
  • （2）替換輸出頭：將LLM原有的頂層線性層（LM Head）替換為全新的線性層（Reward Head）。如《大型模型演算法》1.1.3節所述，LLM的最後一層通常是一個線性投影層（LM Head），其輸入維度為隱藏狀態大小（hidden_size），輸出維度為詞表大小（vocab_size），用於生成對應於全詞表的logits值（機率分佈）。而獎勵模型只需輸出一個分數值，因此，LLM的輸出頭被替換為一個全新的線性層，該層的輸出維度為1（out_dim=1），用於生成獎勵分數。

【RLHF與RLAIF】PPO訓練中四種模型的合作關係

如圖所示，RLHF的第二階段需要載入四個模型進行強化學習訓練。這四種模型的角色和相互關係如下：

  • （1）策略模型（Policy Model, Actor）：RLHF訓練的核心，負責根據輸入的Prompt生成回答At。策略模型需平衡兩個目標：一方面，透過獎勵模型和價值模型引導最佳化以符合人類偏好；另一方面，透過參考模型施加KL約束，限制生成內容與初始行為分佈的偏離，避免過度激進或不穩定的回答。
  • （2）參考模型（Reference Model）：為策略模型提供穩定的行為基準，透過計算策略模型與參考模型輸出之間的KL距離，可以在訓練過程中對策略模型施加約束，防止其過度偏離參考模型的機率分佈。這樣可以避免策略模型偏離SFT模型太遠，確保生成行為的穩定性，從而有效防止策略模型因過度追求獎勵而出現“獎勵欺騙”現象。
  • （3）獎勵模型（Reward Model, RM）：為策略模型生成的回答分配即時獎勵Rt，模擬人類偏好反饋。獎勵模型與價值模型協作，共同引導策略模型朝著人類偏好的方向最佳化。如《大型模型演算法》 7.2.1節所述，在RLHF訓練中，有時還會結合多個獎勵模型以增強效果。
  • （4）價值模型（Value Model, Critic）：全面評估策略模型生成的回答，預測在特定狀態下執行某動作的長期回報，計算價值估計Vt。

【RLHF與RLAIF】PPO訓練中四種模型的結構與初始化（通用&低成本方案）

通常，在通用&低成本場景下，可以簡化模型設計，RLHF涉及的四種模型擁有類似的結構，主要區別在於各模型的頭部（Head）。這些模型可以基於SFT模型（經過有監督微調後的LLM版本）進行改造和初始化，如圖所示：

  • （1）策略模型：結構與SFT模型完全一致，包含N層Transformer Decoder和LM Head（Language Modeling Head）。在引數初始化時，通常直接複製SFT模型的所有引數。在隨後的PPO訓練中，策略模型的引數會持續更新。
  • （2）參考模型：結構與SFT模型一致，包含N層Decoder和LM Head。引數初始化時，同樣直接複製SFT模型的所有引數。但在後續的PPO訓練中，參考模型的引數始終保持凍結（不更新）。因此，在PPO訓練的初始時刻，參考模型與策略模型完全一致，在策略模型的最佳化過程中，參考模型為策略模型提供了一個穩定的行為參考基準。
  • （3）獎勵模型：與SFT模型的底層Decoder部分結構一致，但將頭部結構替換為Reward Head（獎勵頭）。在引數初始化時，僅複製SFT模型的底層Decoder（N層），而Reward Head的引數通常隨機初始化。在隨後的PPO訓練中，獎勵模型的引數始終保持凍結（不進行更新）。
  • （4）價值模型：結構類似於獎勵模型，包含N層Decoder和Value Head（價值頭）。在引數初始化時，可以複製獎勵模型的所有引數，有時也可基於SFT模型進行改造。在隨後的PPO訓練中，價值模型的引數會持續更新。

  • （2）價值向量：形狀為（3, 1）的向量，表示每個Token的價值估計。

【RLHF與RLAIF】RLHF：四種模型可以共享同一個底座

  • 在RLHF中，透過共享同一個底座（N層Decoder），可以顯著降低視訊記憶體開銷。這是因為參考模型、獎勵模型、策略模型和價值模型之間具有較高的相似性。採用LoRA微調技術（詳見《大型模型演算法》 第2.1.2節），四種模型可以共用同一底座，同時保持底座引數凍結（不更新）。
  • 具體而言，參考模型僅用於評估對比，不參與訓練；獎勵模型在RLHF的第一階段（獎勵建模）中進行訓練；而策略模型和價值模型則在第二階段（策略最佳化）中進行訓練。因此，如圖所示，為獎勵模型、策略模型和價值模型分別配備了獨立的LoRA模組（引數），透過組合相應的LoRA模組、Head和共享的底座，即可動態組裝所需的模型。在訓練過程中，僅更新LoRA模組和Head部分的引數。
  • 四種模型共享相同的底座（N層Decoder）。該底座引數常駐GPU並保持凍結狀態，而LoRA模組和Head部分則根據PPO訓練的不同環節，可動態載入到GPU中或動態swap。

【RLHF與RLAIF】PPO訓練中各模型的輸入與輸出

  • 涉及的四個模型具有不同的輸入與輸出（圖中未展示等特殊Token），總結如表7.1所示。
  • 所有四個模型中，只有策略模型執行了生成過程。策略模型根據輸入的Prompt（或上下文）生成了三個Token，即三個動作（[你]、[先去]、[。]）（動作與狀態的建模見《大型模型演算法》圖7.1）。其餘三個模型（參考模型、獎勵模型、價值模型）均以Prompt和策略模型生成的回答為基礎，執行相應的計算：
  • （1）參考模型：輸出回答對應的對數機率（LogProbs），與策略模型輸出的對數機率計算出KL距離，確保策略模型不會偏離基準分佈過遠。
  • （2）獎勵模型：對生成的回答（策略模型的動作）進行逐Token的獎勵打分，生成即時獎勵Rt。如7.2.2節所述，通常只使用最後一個Token對應的獎勵分數，即與 Token對應的獎勵分數。
  • （3）價值模型：對生成的回答（策略模型的動作）進行逐Token的價值估計，用於評估每個動作的長期回報Vt。

【RLHF與RLAIF】PPO訓練中KL距離的計算過程

  • 在TRL等框架的實現中，KL距離的作用體現在對Advantage的計算環節，透過懲罰獎勵值來達到約束目的。如《大型模型演算法》6.2.3節所述，在基於GAE演算法計算Advantage時，需要用到獎勵模型估計的獎勵分數。在此基礎上，可以對獎勵分數進行調整。
  • 如圖所示，策略模型和參考模型均會輸出Logits，並進一步轉換為對數機率（LogProbs），轉換公式可參考《大型模型演算法》第2.2.5節，基於這些對數機率即可完成KL距離的計算。

  • （1）策略模型生成回答Token：以Prompt（或上下文）作為策略模型的輸入，生成回答（response）以及對應的Logits，response包括三個Token：[你]、[先去]和[。]。注意，在實際RLHF訓練中，該步驟已在經驗採集（Make Experience）環節完成。
  • （2）計算對數機率：在步驟（1）中，策略模型已經生成了Logits，即一個形狀為（序列長度=3，詞表大小=7）的矩陣。進一步，可以將該矩陣轉換為對數機率（LogProbs）矩陣。
  • （3）參考模型生成對數機率：以Prompt和response作為輸入，參考模型推理並生成Logits，擷取對應於response部分的Logits，進而可得到參考模型的對數機率矩陣。
  • （4）抽取（Gather）：以步驟（1）中策略模型生成的三個回答Token為索引，在策略模型和參考模型的對數機率矩陣中，分別提取出對應Token的對數機率值。透過這一操作，可以得到三個Token的對數機率值——策略模型的logprob和參考模型的ref_logprob。全詞表範圍下的對數機率矩陣形狀為（序列長度=3，詞表大小=7），而抽取後的logprob和ref_logprob形狀為（3，1）。
  • （5）KL距離計算：如圖所示，對logprob和ref_logprob進行逐位置相減，最終得到策略模型和參考模型之間的KL距離。

【RLHF與RLAIF】拒絕取樣（Rejection Sampling）微調

  • 拒絕取樣(Rejection Sampling)微調：是一種基於資料篩選的模型微調策略，旨在透過人工或模型標註對模型生成的樣本進行篩選，剔除低質量的樣本，僅保留優質樣本用於進一步微調，Anthropic、Meta等公司均曾利用該技術對語言模型進行最佳化。
  • 該過程可重複多輪進行。

【RLHF與RLAIF】RLAIF與RLHF的區別

  • RLAIF（Reinforcement Learning from AI Feedback，基於AI反饋的強化學習）是一種以AI反饋為基礎的強化學習方法。該方法興起於2022年，可作為RLHF的一種替代方案。
  • RLAIF與RLHF在整體流程上具有較高的相似性，如圖所示，其主要區別在於偏好樣本的標註環節。RLHF依賴人類進行偏好標註，而RLAIF則透過AI完成此過程。

【RLHF與RLAIF】Claude模型應用的憲法AI（CAI）原理

  • CAI（Constitutional AI, 基於憲法的AI）是一種用於大型模型訓練的RLAIF型別的演算法，由美國AI獨角獸Anthropic公司於2022年提出[62]，併成功應用於Claude、DeepSeek等模型的訓練。
  • 該演算法以一套預先制定的憲法原則為指導，透過結合AI反饋和自我批評機制，得以更加高效地提升模型的有用性和無害性。 CAI的訓練過程：
  • （1）自我批評與回答修訂：初始模型（例如RLHF模型）根據輸入的Prompt生成回答，隨後依據隨機選取的一條憲法原則對回答進行批評與修訂。模型可以自動識別回答中可能存在的潛在有害、不道德或不合規內容，並根據批評結果對回答進行自我修訂。該過程可以多次迭代，透過逐步最佳化，使生成的回答更加符合預期。
  • （2）訓練監督學習模型（SL-CAI）：修訂後的回答與人類標註的有用性樣本共同組成監督學習資料集，用於訓練監督學習模型（SL-CAI）。透過對預訓練語言模型進行微調，SL-CAI能夠生成更貼近目標分佈的回答，從而顯著降低後續強化學習階段的探索成本。
  • （3）生成回答與AI標註：使用SL-CAI模型針對相同的Prompt生成多組候選回答，採用獨立的反饋模型（例如預訓練模型）對這些候選回答進行AI標註。反饋模型會基於隨機選取的一條憲法原則為回答打分。此階段的AI標註機制有效降低了對人工標註的依賴。
  • （4）訓練獎勵模型：將AI標註資料與人工標註資料混合，形成偏好資料集，並以此訓練獎勵模型，或稱為偏好模型（Preference Model，RM）。
  • （5）強化學習最佳化目標模型（RL-CAI）：在強化學習階段，利用獎勵模型提供獎勵訊號，透過強化學習演算法（例如PPO）對目標模型進行最佳化。此階段逐步增強模型在無害性和有用性上的表現，同時減少模型在敏感問題上的逃避傾向，確保其行為更加透明和可靠。

【RLHF與RLAIF】OpenAI RBR（基於規則的獎勵）

  • RBR（Rule Based Rewards，基於規則的獎勵）是一種基於規則的偏好建模方法，由OpenAI於2024年提出。
  • 該方法透過人工標註資料訓練出一個RBR獎勵模型，將AI反饋融入傳統的RLHF流程中。

【邏輯推理能力最佳化】ORM和PRM（結果獎勵模型和過程獎勵模型）

  • 結果獎勵模型（Outcome Reward Model, ORM）：對於給定的Prompt及模型生成的回答y，結果獎勵模型僅對最終輸出的結果進行整體驗證和獎勵評分。

【邏輯推理能力最佳化】MCTS的執行過程

【LLM基礎拓展】常規訓練時前向和反向過程

這張圖展示了神經網路在常規訓練中的前向傳播（Forward）和反向傳播（Backward）流程：

  • 上半部分是前向傳播：輸入依次透過各層（Layer 1~4），每層的啟用值（Activation）會被快取，用於後續計算梯度。
  • 下半部分是反向傳播：從最終的損失（Loss）開始，逐層計算梯度（Gradient 4 → 1），並利用前向快取的啟用值進行鏈式求導。

【LLM基礎拓展】梯度累積（Gradient Accumulation）訓練

這張圖對比了常規訓練與梯度累積（Gradient Accumulation）訓練的差異：

  • 常規訓練（上方）：每個 batch（如 Batch 1）執行一次前向 & 反向傳播，立刻計算梯度並更新模型引數，更新頻繁，對視訊記憶體要求較低，但 batch size 不能太大。
  • 梯度累積訓練（下方）：多個 batch（如 Batch 1~3）分別計算梯度，不立刻更新，而是將多個梯度累加後統一更新一次，等效於更大的 batch size，適合視訊記憶體受限的情況。

【LLM基礎拓展】Gradient Checkpoint（梯度重計算）

這張圖對比了常規訓練與梯度重計算（Gradient Checkpoint）訓練的不同：

  • 常規訓練（上方）：每層的啟用值（Activation）都完整儲存，便於反向傳播時直接使用，但視訊記憶體佔用大，不適合大型模型。
  • 梯度重計算（下方）：只儲存關鍵層的啟用值（如 Activation 1 和 3），在反向傳播時，對中間未儲存的啟用值（如 2 和 4）進行重新前向計算，以此節省視訊記憶體。
  • 總結：梯度重計算（Gradient Checkpointing）透過以多算換空間的方式降低視訊記憶體使用，適合訓練大型模型時節省資源，但代價是訓練時間略有增加。

【LLM基礎拓展】Full recomputation(完全重計算)

  • 完全重計算（Full Recomputation），即在反向傳播階段不保留任何啟用值，而是每次需要時從輸入開始重新執行前向計算獲取中間啟用。
  • 這種方式極大地節省了視訊記憶體，非常適合在記憶體受限場景下訓練大型模型，但會顯著增加計算量和訓練時間。

【LLM基礎拓展】RNN（Recurrent Neural Network）

  • RNN（Recurrent Neural Network）：一種專門處理序列資料的神經網路，透過迴圈連線在每個時間步保留並更新前一時刻的隱藏狀態，使網路具有記憶能力。
  • 優點：結構簡單、能夠捕捉短期依賴；缺點：易出現梯度消失/爆炸，難以學習長距離依賴。

【LLM基礎拓展】Pre-norm和Post-norm

  • Pre-norm：在子層（如自注意力或前饋網路）輸入前先執行LayerNorm，然後經過子層並與原始輸入進行殘差連線，可提升梯度流動和深層模型的訓練穩定性
  • Post-norm：在子層輸出後與原始輸入進行殘差連線，再執行LayerNorm，是Transformer的經典結構，但在深層網路中可能導致梯度衰減和訓練不穩定

【LLM基礎拓展】BatchNorm和LayerNorm

  • BatchNorm（批次歸一化）：對每個小批次中的每個通道進行標準化（減去均值、除以標準差），再加上可學習的縮放和偏移引數。
  • LayerNorm（層歸一化）：對每個樣本的所有特徵維度進行標準化（減去均值、除以標準差），同樣使用可學習的縮放和偏移引數。

【LLM基礎拓展】RMSNorm

  • RMSNorm（Root Mean Square Layer Normalization）是一種歸一化方法，僅基於輸入特徵的均方根值（RMS）進行規範化，不計算均值偏移。
  • 對每個樣本的特徵向量 (x)，先計算 (\mathrm{RMS}(x)=\sqrt{\tfrac{1}{d}\sum_i x_i^2})，再做縮放：(\hat{x}=x / \mathrm{RMS}(x)\times \gamma + \beta)。
  • 相較於 LayerNorm，它省略了減均值步驟，計算更高效且在多種 Transformer 架構中表現相近或更優。

【LLM基礎拓展】Prune（剪枝）

  • 模型剪枝（Model Pruning）是一種壓縮深度神經網路的方法，透過移除冗餘或不重要的權重、神經元或通道，減少模型引數量和計算量。
  • 可以在儘量保持原始精度的前提下，提升模型的推理速度、降低記憶體佔用。
  • 典型流程包括：評估重要性 → 標記待剪引數 → 重訓練（Fine-tuning）→ 驗證效能。

【LLM基礎拓展】溫度係數的作用

  • 在生成階段，溫度係數 (T) 透過放縮模型輸出的 logits 分佈來控制取樣多樣性。
  • 當 (T < 1) 時，分佈“尖銳”，模型更傾向於選擇機率最高的 token，回答更固定。
  • 當 (T > 1) 時，分佈“平坦”，取樣結果更具隨機性。
  • 適當調整溫度可以在準確性和創造性之間取得平衡——低溫度提升確定性，高溫度增強多樣性。

【LLM基礎拓展】SwiGLU

  • 基本結構：SwiGLU 是 Gated Linear Unit（GLU）的變體，將輸入向量透過兩條線性對映，一路直接輸出，一路經過 SiLU（Swish）啟用後作為門控訊號，兩者逐元素相乘得到最終輸出。
  • 啟用優勢：用 SiLU（Swish-1）替換原 GLU 中的 sigmoid，使門控更平滑、梯度更穩定，有助於加速訓練收斂並提升下游任務表現。

【LLM基礎拓展】AUC、PR、F1、Precision、Recall

被面試者、工程師經常混淆的演算法常用指標，我畫了一張圖來對比總結，區別一目瞭然。

  • AUC（Area Under the ROC Curve）：衡量二分類模型在所有可能閾值下區分正負樣本的能力，ROC（Receiver Operating Characteristic）曲線以假正例率（FPR）為橫軸、真正例率（TPR）為縱軸，AUC 值越接近 1 越好；若 AUC≈0.5 則相當於隨機猜測。
  • PR Curve（Precision–Recall Curve）：模型在不同召回率（Recall）下的精確率（Precision）變化趨勢。可透過曲線下面積來綜合評估模型對正類的檢出能力。
  • Precision（精確率）：在所有被預測為正例的樣本中，實際為正例的比例，衡量誤報率高低，適用於誤報成本高的場景，個人認為“查準率”的翻譯更貼切。Precision = TP / (TP + FP)
  • Recall （召回率）：在所有真實為正例的樣本中，被正確預測為正例的比例，衡量漏報率高低，適用於漏報成本高的場景，個人認為“查全率”的翻譯更貼切。Recall = TP / (TP + FN)
  • F1 Score （F1 值）：Precision 與 Recall 的調和平均，當兩者平衡時取得最高分。F1 Score = 2 * (Precision * Recall) / (Precision + Recall)
  • Accuracy （準確率）：在所有樣本中，預測正確的比例，適用於類別分佈較為均衡時的整體效能評估。Accuracy = (TP + TN) / (TP + TN + FP + FN)
  • BLEU（Bilingual Evaluation Understudy；機器翻譯評估指標）：基於 n-gram 重疊率評估翻譯結果與參考的相似程度。
  • ROUGE（Recall-Oriented Understudy for Gisting Evaluation；摘要評估指標）：生成摘要與參考在 n-gram、最長公共子序列或詞重疊方面的匹配情況，關注召回效能。

【LLM基礎拓展】RoPE位置編碼

  • RoPE，Rotary Position Embedding，旋轉位置編碼，由蘇劍林提出
  • 原理：對 Transformer 中的 Query 和 Key 隱藏向量應用旋轉變換（基於正弦、餘弦函式），將位置資訊編碼為向量的相位差。
  • 相對位置：旋轉操作天然地保留了向量之間的相對角度差異，使模型能夠捕捉相對位置資訊而不依賴絕對位置索引。
  • 優點：無需額外引數、計算開銷低；在長序列場景下相較於傳統絕對位置編碼具有更好的泛化和效能。
